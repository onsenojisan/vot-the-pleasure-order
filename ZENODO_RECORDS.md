# Zenodo Records

This file is the public index of archived records for VOT / The Pleasure Order.

Zenodo is the authoritative archive for fixed release packages. GitHub is used as a public landing page and routing layer.

This ledger consolidates the public Zenodo records associated with VOT / The Pleasure Order, the O-M-A validation program, the L4 structural-proxy program, and the older Omega-facing materials.

It is intended for public routing and claim-boundary clarity. It does not add any empirical claim.

For the current public status of the theory and its claim boundary, see [Current Status](CURRENT_STATUS.md). The records below are archival evidence/design records and do not by themselves establish VOT as a validated theory.

## Status Labels

| Label | Meaning |

## Public Claim Boundary

The current public record supports only bounded claims:

- O-M-A records support exploratory/proxy D2/D2+ evidence and DC2/D3 measurement design.
- L4 supports rule-conditional structural-proxy evidence under fixed gates.
- Older Omega records are historical, conceptual, methodological, or navigational unless explicitly reauthorized under the boundary conditions.

The public record does not currently support claims that:

- Omega is validated.
- Omega predicts collapse.
- The Pleasure Order is empirically proven.
- O-M-A has been directly validated as a distinct measurement architecture.
- Pleasure, suffering, or directionality has been directly measured as the theory defines them.

## File Location Policy

- Zenodo holds release files, PDFs, zip packages, manifests, QA outputs, checksums, and metadata.
- This GitHub repository holds public routing, citation metadata, and claim-boundary documents.
- Private working repositories hold drafts, generated outputs, raw caches, exploratory scripts, and internal audit trails.

## Maintenance Rules

When a new Zenodo record is added:

1. Add the record DOI and, if applicable, concept DOI.
2. Assign exactly one status label.
3. State the claim boundary in one sentence.
4. If the record is historical or superseded, say so in the table.
5. If the record contains empirical results, state whether the labels/outcomes were independent and whether the design was preregistered or post hoc.
6. Do not remove deleted or superseded records if they have been publicly cited; mark them as deleted/superseded instead.
